# AIRL Performance Benchmark: Bytecode→Cranelift JIT

**Date:** 2026-03-23
**Mode:** Release build (`cargo build --release --features jit -p airl-driver`)
**Machine:** Linux 6.8.0-60-generic
**Method:** Best of 3 runs, wall-clock time via `date +%s%N`

## Context

AIRL now has four execution modes:
- **Interpreted** — tree-walking interpreter (`eval.rs`), full contract checking
- **IR VM (`--compiled`)** — AST → IR nodes → Rust VM (`ir_vm.rs`), no contracts, self-TCO
- **Bytecode VM (`--bytecode`)** — AST → IR → flat register-based bytecode → tight execution loop (`bytecode_vm.rs`), self-TCO
- **JIT (`--jit`)** — Bytecode VM + Cranelift native code for eligible functions (`bytecode_jit.rs`)

## Results

| Benchmark | Python 3 | Interpreted | IR VM | Bytecode | **JIT** | Bytecode→JIT |
|-----------|----------|------------|-------|----------|---------|-------------|
| fib(30) | 302ms | 42,006ms | 7,355ms | 4,605ms | **10ms** | **460x** |
| fact(12) x 10K | 52ms | 829ms | 161ms | 163ms | **6ms** | **27x** |
| sum-evens x 5K | 52ms | 3,497ms | 1,175ms | 835ms | **854ms** | ~1x |

## Analysis

**fib(30): 460x speedup, 30x faster than Python.** The `fibonacci` function is pure-int with no closures, lists, or builtins — fully JIT'd to native x86-64. Cranelift eliminates all Value boxing, HashMap lookups, call frame allocation, and per-instruction dispatch. Self-recursive calls compile to direct Cranelift `call` instructions with the `i64` calling convention. The 10ms includes program startup, stdlib loading, JIT compilation, and execution.

**fact(12)x10K: 27x speedup.** `fact-helper` is a tail-recursive int function — the JIT compiles TailCall to a loop-back jump (no stack growth). 10K iterations of `fact(12)` in 6ms total.

**sum-evens x5K: ~1x (not eligible).** Uses `map`/`filter`/`fold` (closures) and lists — these opcodes (`MakeClosure`, `MakeList`, `CallReg`) make the function ineligible for JIT. Falls back to bytecode transparently. Phase 2 could add runtime helper calls for these operations.

## Fixture Compatibility

All 26 valid fixture tests produce identical output in `--jit` mode vs interpreted mode. The JIT is transparent — ineligible functions fall back to bytecode, which falls back to builtins.

## What Gets JIT'd

With `AIRL_JIT_DEBUG=1`, the JIT reports which stdlib functions it compiles:
- **Compiled:** `pow`, `min`, `max`, `clamp`, `even?`, `odd?`, `sign`, `abs`, `gcd`, `lcm` (pure-int math functions)
- **Skipped:** `map`, `filter`, `fold`, `sort`, `reverse`, `concat`, etc. (use lists/closures)
- **Skipped:** All string/map functions (use builtins)

User functions are compiled if they only use primitive operations (arithmetic, comparisons, control flow, calls to other JIT'd functions).
