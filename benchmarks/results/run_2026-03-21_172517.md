#### Task 05: String Tokenizer

```lisp


`★ Insight ─────────────────────────────────────`
AIRL's `split` builtin does literal delimiter matching, so splitting `"hello  world  foo"` by `" "` produces empty strings between consecutive spaces. We need `filter` with a predicate to remove empties — AIRL's stdlib provides `is-empty-str` but we can also compare with `""` directly using `!=`. The `filter` builtin takes `(fn [x] condition) list` order.
`─────────────────────────────────────────────────`

(defn tokenize
  :sig [(s : String) -> List]
  :requires [(valid s)]
  :ensures [(valid result)]
  :body (filter (fn [tok] (!= tok "")) (split s " ")))

(print (tokenize "hello  world  foo"))
(print (tokenize ""))

`★ Insight ─────────────────────────────────────`
1. **Why `filter` over empty strings?** AIRL's `split` is a thin wrapper around Rust's `str::split()`, which preserves empty segments between consecutive delimiters — so `"hello  world"` split by `" "` yields `["hello", "", "world"]`. Filtering with `(!= tok "")` cleans these up.
2. **Contract minimalism:** Since `tokenize` accepts any string (including empty), the weakest meaningful precondition is `(valid s)`. The postcondition could assert `(>= (length result) 0)` but that's a tautology for lists, so `(valid result)` is the pragmatic choice here.
```
